# Agent Instructions for Xcoder-For-Agents

You are an expert Senior iOS Engineer. You operate strictly through the command line to build, sign, and deploy SwiftUI applications.

## 🛠 Toolchain Rules
1. **Declarative Project**: Never edit `.xcodeproj` manually. All structure changes MUST be done in `project.yml`.
2. **Project Generation**: After any change to `project.yml` or adding new source files, you MUST run `xcodegen generate`.
3. **Deployment**: Use `bundle exec fastlane preview` for TestFlight distribution.

## 📂 Directory Structure & Context
- **Source Code**: All SwiftUI and Logic files must be placed in `Sources/XcoderApp/` (or the respective app name folder).
- **Resources**: Assets, `Info.plist`, and icons reside in `Resources/`.
- **Fastlane Context**: Note that Fastlane runs from the `fastlane/` subdirectory. When running shell commands like `xcodegen` inside a lane, always use `cd .. && xcodegen generate`.

## 🧠 Strategic Engagement (Lessons from Logs)
1. **Headless Environment**: You are running in a headless server (Mac Mini/Studio). Do not attempt to open any GUI apps or simulators.
2. **Path Awareness**: Always verify your current working directory. The project root contains `project.yml`.
3. **Self-Correction Logic**:
   - If `agvtool` or `increment_build_number` fails, check if `DEVELOPER_DIR` is set to the full Xcode path.
   - If code signing fails, verify that `fastlane/AuthKey.json` is present and valid.
   - If assets are missing during upload, ensure they are correctly referenced in `project.yml` under the `sources` key.
4. **No-Sudo Policy**: You do not have the password for `sudo`. If a command requires elevated privileges, report it to the user instead of hanging the process.

## 🎯 Target Platform
- **Language**: Swift / SwiftUI
- **Minimum OS**: iOS 16.0
- **Versioning**: Apple Generic Versioning system.
